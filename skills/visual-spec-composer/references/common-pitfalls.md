# 常见陷阱与规避

本文档列出使用 visual-spec-composer 时的常见错误及规避方法。

## 访谈阶段

### 陷阱 1：跳过访谈直接生成

**现象**：拿到需求就直接写 VGS，省略澄清步骤

**后果**：产出与用户预期不符，返工成本高

**规避**：即使需求看似明确，也至少完成 2 轮核心澄清（主体 + 风格）

### 陷阱 2：访谈问题过于笼统

**现象**：问"你想要什么风格"而不提供选项

**后果**：用户无法有效回答，对话陷入循环

**规避**：每个问题附 2-3 个推荐答案，降低决策成本

### 陷阱 3：未确认方向就生成 VGS

**现象**：访谈结束后直接输出 VGS，跳过 Human Brief 确认

**后果**：方向错误导致 VGS 全部作废

**规避**：Step 2.5 必须输出 Human Brief 并等待确认

## VGS 填充阶段

### 陷阱 4：参考图 ownership 缺失

**现象**：有参考图但未声明借用范围

**后果**：生成结果混淆了不应借鉴的元素

**规避**：每张参考图必须明确 ownership（style / composition / subject）

### 陷阱 5：风格词堆砌

**现象**：style 字段写入大量互相矛盾的风格词

**后果**：模型无法解析，生成结果混乱

**规避**：主风格词 ≤ 2 个，辅助修饰 ≤ 3 个

### 陷阱 6：忽略复杂度评估

**现象**：所有任务都选 one_shot

**后果**：复杂任务一次生成质量差

**规避**：block_count > 4 时必须评估 two_stage 可能性

## 输出阶段

### 陷阱 7：Render View 与 VGS 不一致

**现象**：Full Render View 描述内容与 VGS 字段值矛盾

**后果**：按 Render View 生成的结果违反 VGS 规格

**规避**：生成 Render View 后做一致性自检

### 陷阱 8：未提供降阶方案

**现象**：只输出最优方案，无 fallback

**后果**：最优方案失败后无法恢复

**规避**：execution.fallback 字段必须填写具体降阶策略

## 语言与格式

### 陷阱 9：产出语言与用户不一致

**现象**：用户用中文提问，产出全英文

**后果**：用户理解成本增加

**规避**：未指定语言时，默认使用中文产出（VGS 结构字段名保持英文）

### 陷阱 10：VGS YAML 格式错误

**现象**：缩进错误、引号遗漏、特殊字符未转义

**后果**：下游解析失败

**规避**：使用 `scripts/validate_vgs.py` 校验后再交付
